# CoachCast

A web app for independent youth sports coaches. After a session, the coach records
or types a quick note; CoachCast generates a parent-ready progress recap in the
coach's chosen voice, which the coach reviews, edits, and sends. Recaps are logged
to a per-athlete progress timeline. Multi-sport: basketball, golf, tennis, baseball,
soccer.

The recap is generated by Claude through a **serverless function** — the Anthropic
API key lives on the server and is never shipped to the browser.

## Stack

- Vite + React (JavaScript)
- Vercel serverless function for the Anthropic call (`api/generate-recap.js`)
- `lucide-react` for icons
- `localStorage` for phase-1 persistence (one coach, one device)
- Deploy target: Vercel

## Run it locally

```bash
npm install
cp .env.example .env.local      # then put your real key in .env.local
npm run dev                     # http://localhost:5173 — UI *and* /api work
```

`npm run dev` serves the API via a dev-only Vite middleware that reuses the exact
serverless handler in `api/generate-recap.js`, so the full record → generate →
review → send flow works without extra tooling. You can also use the Vercel CLI if
you prefer an environment identical to production:

```bash
npm i -g vercel
vercel dev
```

Sign in with any email and a 6+ character password — auth is a demo stub in phase 1.

## Deploy (Vercel)

```bash
vercel env add ANTHROPIC_API_KEY     # add the key to the Vercel project
npm run build
vercel --prod
```

## Where the key lives

`ANTHROPIC_API_KEY` is read only by `api/generate-recap.js` (server-side) and the
dev middleware in `vite.config.js` (also server-side). It is **never** prefixed with
`VITE_`, never imported into a client file, and never bundled. `.env.local` is
gitignored.

## Architecture seams (so phase 2 drops in cleanly)

- **Auth** — `src/auth/AuthContext.jsx` exposes `{ user, signIn, signOut }`. Phase 1
  is an in-memory demo; phase 2 replaces only this file's internals (e.g. Clerk or
  Supabase Google sign-in). Components keep calling `useAuth()`.
- **Data** — `src/data/store.js` owns every read/write (currently `localStorage`).
  Phase 2 moves these to `/api/*` functions querying Postgres keyed by `coach_id`,
  touching only this file.
- **AI call** — `src/lib/api.js` is the only client code that hits the recap
  endpoint, and it calls our own `/api/generate-recap`, never `api.anthropic.com`.

## Project layout

```
api/generate-recap.js     serverless Anthropic call (key server-side)
src/
  App.jsx                 root: auth gate + web shell
  auth/AuthContext.jsx    auth seam
  data/store.js           data seam (athletes + recaps, localStorage)
  lib/                    sports/tokens config, api client, hooks, formatting
  components/             Auth, WebNav, Phone, PhoneApp, Home, AthletePage,
                          Capture, Review, Sent, shared ui + Screen
  styles.jsx              global <Style>: fonts, resets, keyframes
  main.jsx
```
